# Galgame Character Skills

一个简单的生成器。给定一段文本（Galgame剧情脚本/小说/任何单个长文本文件），自动将其中一个角色蒸馏为skills或是适用于SillyTavern的角色卡。

支持使用VNDB相关信息进行信息增强。在提供VNDB信息时，生成角色卡将会自动使用对应的立绘合并一份.png角色卡。如果你的角色于VNDB的信息不全/有偏差，最好不要使用。切勿在使用非VN角色时使用，哪怕你的目标是具有视觉小说版本的同名角色。

最好自行对生成结果进行一定程度的调整或是检查，即使有VNDB提供的信息也不保证符合实际。尤其是那些不属于VN的角色。

只测试过几个Galgame剧情文本，一般来说不要把多路线的所有内容一股脑塞进来，是没问题的。

理论上来说你也可以使用QQ聊天记录，或是任何其他东西来生成。

## 安装

```bash
pip install -r requirements.txt
```

## 使用

```bash
python main.py
```

启动后自动打开浏览器访问 `http://127.0.0.1:5000`。

## 配置 LLM

在 Web 界面中填写：
- **Base URL**: API 地址，如 `https://api.openai.com/v1` 或 `http://localhost:11434`
- **Model Name**: 模型名称，如 `gpt-4` 或 `ollama/llama2`
- **API Key**: 你的 API 密钥

然后选择运行模式与对应文件开始处理。

## 注意事项

分析的文本一定要像人写的。像是雪人语啊，还有一些写得不太像人类的小说啊，不保证效果。据称，使用樱花萌放的文本出来的完全就是一个省略号机器。

确保你分析的文本中对应的角色每次登场时，别人叫他是用本名或是简称，而不是任何完全无关的外号，如有这种情况需要附加指令以添加额外信息。

对于galgame文本，尽可能清理其中所有的引擎指令，其他无关人员的主要线路，所有的h场景，与绝对明显冲突的内容。

例如，完全丢弃其它任何你的角色没有直接登场/间接提到的场景。完全丢弃无意义的场景，丢弃不同线路的结局，丢弃非主角视点的场景等。

对于长篇小说或是其他内容，确保至少上下文看起来是一个连贯的故事。绝对不能使用写得太过谜语人的小说，像是各种频繁穿越流的，视点不统一的，或者单纯写得确实很烂的等等。

如果你的文本中间有任何与你的角色关联性不大的长内容，如果你认为这完全无必要，最好完全丢弃它。如果你认为有一定必要，请使用其他方法归纳总结，或是手动注入附加指令/手动撰写一份其他文件。这个工具不适合用来处理这些很琐碎的内容。

实际运行时长可能在10-15分钟或是更久，取决于API的返回速度，你使用的模型，并发数量，文本长度，压缩策略等等。

## 已知问题

如果使用Gemini系列模型，有概率无法绕过其安全设置。这个，没什么办法。确保你使用的文本不包含任何可能敏感的内容。
